如果你的節點是10/1以後才安裝的，會卡在這裡轉圈圈，解決方法如下

 

結束節點程式，由File --> Exit，不是按右上的X喔

 

(下面的步驟其實可以略過，只不過手動下載image檔後，待會兒可以比較快看到建立Container的結果，不用再等)

手動下載image檔，開啟命令提示字元(cmd)，執行

docker pull pinetwork/pi-node-docker

 

到Docker確定image下載完成

 

找到user-preferences.json，不同的作業系統的路徑不一樣

l   Windows 10是位於 %appdata%\Pi Network\

l   MAC檔案在 /Users/使用者名稱/Library/Application Support/Pi Network/

 

不知道 %appdata% 是什麼的人，請看這篇 https://yuanrui919.github.io/mvappdata/ 。

 

用記事本或純文字編輯軟體開啟它，內容如下

{"windowWidth":800,"windowHeight":600,"uuid":"xxxxxxxx-xxxx-xxxx-xxxxxxxxxxxxxxxxx","switchStatus":true,"dockerImage":"pinetwork/pi-node-docker:latest"}

 

在最後的 } 前加上

,"containerFlavor":"stellar","testnetResetCount":"2","stellarConfigVersion":"0.0.9","testnetResetLatestNodeUIVersion":"0.4.4","horizonStatusPreference":false

 

注意，這檔案是JSON格式。頭尾用大括號 {} 包起來的是物件(object)，物件使用 "Key":"Values" 的方式儲存，物件跟物件之間用逗號隔開。如果你打錯字或漏了什麼符號，可能導致格式不正確，請再次檢查。

 

重新執行節點程式，並觀察Docker，Containers會多出一個pi-consensus，注意看它的Log，等它出現「setllar-core entered RUNNING state」這一行

 

關掉節點小紅點，結束節點程式。

 

檢查%appdata%\Pi Network\user-preferences.json，最後應該多出一段

,"stellarSecretSeed":"XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"

 

再重新啟動節點程式，一切就正常了。

 

千萬別聽信奇怪的方法，去手動修改user-preferences.json內的stellarSecretSeed。

 

最後說明一下原因，其實只要加 "containerFlavor":"stellar" 就可以，但testnetResetCount、stellarConfigVersion、testnetResetLatestNodeUIVersion這些值不會自動產生，所以還是手動加。

(先鋒善逸補充：另外三個是運行很久的人才會有的設定，自行添加也沒差，那個紀錄正確的是在官方那)

 

在2022/3/14之前，只有少數人被允許執行pi-consensus容器，其他人都只能跑Stellar-Dummy。那時大部分人的 "containerFlavor" 的值是 "dummy"，而天選之人的值是 "stellar"。

 

從2022/3/14以後，全部架設節點的人都能獲得pi-consensus容器，所以大家的 "containerFlavor" 的值都改成了 "stellar"。

 

但從2022/10/1起，官方不知道在幹嘛，可能某個服務連不上，新節點的user-preferences.json無法修改containerFlavor的值，造成節點程式不知道要啟動哪個Container，所以就一直轉圈圈。
